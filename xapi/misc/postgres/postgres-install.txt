# schema ownership 

The best way to handle this without running commands row-by-row is to log in 
directly as the xapi_user when creating your database objects. 
In PostgreSQL, whoever creates a table or sequence automatically becomes 
its absolute owner.

For java apps, we would like to create all tables with xapi_user ownership. 
The recommened method to do that is to connect to xapi_db as xapi_user. for 
any postgres leftover privileges, 
$ psql -U xapi_user -d xapi_db

For any mixed ownership databases, log in as postgres user and tranfer ownership 
to the desired user. 


-- Connect to the target database
$ psql -U postgres
\c xapi_db

-- 1. Transfer ownership of the entire public schema to your user
ALTER SCHEMA public OWNER TO xapi_user;

-- 2. Re-assign ownership of all existing tables globally
REASSIGN OWNED BY postgres TO xapi_user;

-- 3. Drop any leftover superuser dependencies in this specific database
ALTER DEFAULT PRIVILEGES FOR ROLE postgres IN SCHEMA public GRANT ALL ON TABLES TO xapi_user;
